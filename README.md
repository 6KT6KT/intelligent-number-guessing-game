智能猜数字游戏（Intelligent Number Guessing Game）

一、项目介绍
1. 任务定义（符合作业 “任务定义清晰” 要求）
问题背景：响应 “人工智能基础” 作业 3“基于大模型的无代码 / 低代码智能应用实践” 要求，聚焦 “AI 协作开发” 与 “工程化意识” 培养，开发一款功能完整、高兼容的控制台小游戏。
目标用户：编程入门学习者、休闲游戏爱好者，以及需完成课程作业的学生。
交付成果：可直接运行的 Python 代码（仅依赖内置模块）、完整历史记录持久化功能、规范 README 文档（含测试样例与 AI 迭代记录）。
2. 核心价值
践行 “AI 协作” 理念：通过大模型多轮迭代优化，从基础框架到功能完善，体现 “人机协作解决问题” 的能力；
零门槛使用：无需安装第三方库，全平台兼容，控制台直接运行；
可复现可验证：功能模块清晰、测试样例具体，第三方可按文档快速复现所有功能。

二、环境要求
依赖项	版本要求	说明
Python	3.9 及以上	推荐 3.9/3.10 版本，兼容性最佳，符合 Python 主流教学环境
依赖库	无	仅使用random（随机数生成）、datetime（时间记录）、pickle（持久化存储）3 个 Python 内置模块
运行平台	Windows/macOS/Linux	无平台限制，所有终端 / 控制台均可正常运行

三、运行步骤（符合作业 “表达与复现” 要求）
获取代码：从 GitHub 仓库克隆代码，仓库链接：https://github.com/6KT6KT/-
环境验证：打开终端 / 命令提示符，执行python --version，确认版本≥3.9；
启动游戏：
执行启动命令：python number_guessing_game.py；
操作流程：按控制台提示选择难度→输入数字猜测→游戏结束后可选择查看历史记录 / 重玩 / 退出。

四、功能清单（符合作业 “成果完成度” 要求）
功能模块	详细描述	对应作业目标
难度分级	3 档可选：简单（1-50/10 次机会）、中等（1-100/8 次机会）、困难（1-200/6 次机会），输入错误时强制重新选择	满足 “自定义项目” 的创造性要求，合理控制工作量
三重输入验证	1. 非数字输入：提示 “输入无效，请输入纯数字”，不消耗猜测次数；2. 重复输入：提示 “注意：请勿重复猜测已输入的数字”，不消耗次数；3. 边界值输入：提示 “当前难度数字范围为 X-Y”，不消耗次数	提升项目稳定性，避免程序崩溃，体现工程化意识
历史记录持久化	自动保存每条游戏记录（时间、难度、数字范围、使用次数、胜负结果）至game_history.pkl文件，程序启动时自动加载已有记录	解决 “数据丢失” 问题，增强项目实用性，体现问题抽象能力
历史记录可视化	游戏结束后支持查看历史记录，以表格形式直观展示所有记录，信息清晰易读	提升用户体验，满足 “可展示” 要求
重玩与退出	支持重复游玩（递归调用重启游戏）或直接退出，退出前自动保存最新记录	完善交互逻辑，确保功能完整性
实时结果反馈	猜测后即时提示 “猜测偏小 / 偏大 / 正确”，获胜时显示使用次数，失败时揭示目标数字	核心游戏逻辑完整，符合用户交互预期
